# Save Apply-Ready Updates Instead of Mutating Docs Repositories

The product must solve documentation maintenance, not merely store suggestion status, so a Saved Update will persist the original excerpt, final replacement content, evidence, and Review Decisions for the approved or revised Edit Suggestions. We will not directly modify documentation files or open GitHub pull requests in the take-home version because repository auth, branch management, merge conflicts, and PR review would pull effort away from the AI suggestion quality that the challenge prioritizes.
